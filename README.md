# RIS Conductor 使用者指南

## 這是什麼？

RIS Conductor 是一款自動化放射科報告工具。它能自動從 RIS（放射資訊系統）讀取病人檢查清單、從 PACS（影像存檔系統）擷取影像、呼叫 AI 產生報告草稿，並將結果貼回 RIS 報告編輯器。

簡單來說：**選好病人清單 → 按下開始 → AI 自動產出報告草稿**。

---

## 系統需求

- **Windows 10 或 Windows 11**
- **WebView2 Runtime**（Windows 11 已內建；Windows 10 通常也已安裝。若缺少，[請由此下載](https://developer.microsoft.com/microsoft-edge/webview2/)）
- **網路連線**（AI 報告產生必須連線至 AI 服務商；PACS 若使用 API 方式也需要網路）

---

## 安裝方式

1. 從 GitHub Releases 下載最新版程式
2. 放在任意資料夾中
3. 雙擊執行即可

## 首次設定

第一次啟動時，會顯示設定畫面，請依序填寫：

### 1. 選擇 RIS 系統

| 選項 | 適用醫院 |
|---|---|
| NTUH KReport | 台大醫院 |
| TAH PowerBuilder | 臺安醫院 |

> ⚠️ RIS 系統選擇會影響可用的 PACS 選項——程式會自動隱藏不相容的 PACS 選項。例如：選擇「臺安醫院」時，台大醫院的 PACS 選項會被停用。

### 2. 選擇 PACS 來源

| 選項 | 說明 |
|---|---|
| NTUH PACS (API) | 台大醫院 — 透過 API 擷取影像（推薦，速度較快） |
| NTUH PACS (螢幕擷取) | 台大醫院 — 透過螢幕擷取（需 PACS 視窗保持可見） |
| NTUCC / HCH / BIH / YLH PACS (API) | 台大各分院 |
| TAH PACS (螢幕擷取) | 臺安醫院 — 螢幕擷取（需 PACS 視窗保持可見） |

> ⚠️ 選擇 API 方式時，批次開始前會跳出帳號密碼輸入視窗，請輸入您的 PACS 登入憑證。選擇螢幕擷取方式時，需確保 PACS 視窗已開啟且可見。

### 3. 選擇 AI 服務商與模型

| 服務商 | 說明 |
|---|---|
| OpenRouter | 整合多家 AI 模型的閘道器（推薦，可選擇多種模型） |
| OpenAI | 直接使用 OpenAI API |
| Google Gemini | 直接使用 Google Gemini API |
| Anthropic | 直接使用 Anthropic Claude API |
| HuggingFace | 使用 HuggingFace 端點 |

選擇服務商後，會自動列出可用的模型清單，也可手動輸入模型名稱。

> 💡 **模型選擇建議**：較強的模型（如 GPT-4o、Claude Sonnet）產生的報告品質較好，但費用較高；較快的模型（如 GPT-4o-mini、Gemini Flash）速度較快、費用較低，但報告品質可能略遜。初次使用建議從推薦模型開始，再依需求調整。

### 4. 輸入 API Key

將您從 AI 服務商取得的 API Key 貼入欄位。此金鑰僅儲存於本機，不會傳送至任何第三方。

> ⚠️ 除了申請 API Key 之外，請記得至 AI 服務商網站**購買 Credit（儲值）**。只有 API Key 但沒有 Credit，程式會報錯無法使用。各服務商的儲值方式：
> - **OpenRouter**：前往 [https://openrouter.ai/credits](https://openrouter.ai/credits) 加值
> - **OpenAI**：前往 [https://platform.openai.com/settings/organization/billing](https://platform.openai.com/settings/organization/billing) 加值
> - **Google Gemini**：前往 [https://aistudio.google.com/](https://aistudio.google.com/) 查看免費額度或加值
> - **Anthropic**：前往 [https://console.anthropic.com/settings/billing](https://console.anthropic.com/settings/billing) 加值

### 5. 儲存並繼續

填寫完畢後按下「Save & Continue」即可進入主畫面。

## 如何申請 API Key

### OpenRouter（推薦）

1. 前往 [https://openrouter.ai/](https://openrouter.ai/)
2. 註冊帳號
3. 進入 **Keys** 頁面
4. 點擊 **Create Key**，複製產生的金鑰（格式：`sk-or-v1-...`）

### OpenAI

1. 前往 [https://platform.openai.com/](https://platform.openai.com/)
2. 登入或註冊帳號
3. 進入 **API keys** 頁面
4. 點擊 **Create new secret key**，複製金鑰（格式：`sk-proj-...`）

### Google Gemini

1. 前往 [https://aistudio.google.com/](https://aistudio.google.com/)
2. 使用 Google 帳號登入
3. 點擊 **Get API key**，建立並複製金鑰

### Anthropic

1. 前往 [https://console.anthropic.com/](https://console.anthropic.com/)
2. 註冊或登入帳號
3. 進入 **API Keys** 頁面
4. 建立並複製金鑰（格式：`sk-ant-...`）

### HuggingFace

1. 前往 [https://huggingface.co/](https://huggingface.co/)
2. 登入帳號
3. 進入 **Settings → Access Tokens**
4. 建立 Token（格式：`hf_xxxxxxxxxxxxx`）
5. 若使用 Inference Endpoint，還需準備端點 URL

> ⚠️ API Key 輸入後若出現錯誤，請確認：金鑰是否完整複製（沒有多餘空格或換行）、帳號是否已加值、金鑰是否已過期。

## 病人個資保護

本程式在將資料傳送給 AI 之前，會自動執行以下去識別化處理：

1. **臨床病史去識別化**：從臨床病史文字中移除以下個資欄位：
   - 中文姓名、英文姓名
   - 健保卡號
   - 身分證字號、居留證號
   - 出生日期（民國/西元格式無條件移除；西元四位年日期僅在前後文含出生標記或人口統計資訊時移除）
   - 電話號碼、電子郵件
   - 病歷號碼（已排除 YYYYMMDD 格式的日期，避免將檢查日期誤判為病歷號）
   - 地址

   移除後會以標記替代，例如 `[DE-IDENTIFIED_NHI_CARD]`、`[DE-IDENTIFIED_ADDRESS]` 等，確保 AI 無法取得可辨識的個人資訊。

2. **影像去識別化**：PACS 影像在傳送給 AI 時使用 **JPEG 格式**（非 DICOM 格式），原始 DICOM 標籤中的病人姓名、病歷號、檢查機構等資訊不會包含在傳送的影像中。

3. **影像後處理**：針對 DX/CR 類型的影像，程式會自動套用百分位數視窗調整（肺窗/縱隔窗/骨窗三組預設），提升 AI 判讀準確度。這些處理後的影像同樣不含 DICOM 標籤。

> ⚠️ 去識別化僅針對傳送給 AI 的資料。RIS 系統中的原始資料不受影響。

## 使用方式

### 批次處理流程

1. **載入清單**：點擊「Load Worklist」按鈕，系統會從 RIS 讀取待處理的病人清單
2. **確認清單**：下方表格會顯示所有待處理病人（病歷號、檢查號、姓名、檢查項目）
3. **開始處理**：點擊「Start」按鈕，系統會依序處理每位病人：
   - 從 PACS 擷取影像
   - 傳送給 AI 產生報告
   - 將報告草稿貼回 RIS
4. **查看進度**：進度條即時顯示處理進度，日誌區顯示每筆病人的處理狀態
5. **完成**：處理完畢後會顯示失敗清單（如有）

> ⚠️ **批次處理期間請注意**：
> - **RIS 視窗必須保持可見**，不要最小化或被其他視窗完全遮擋，否則程式無法正確操作 RIS
> - **NTUH 模式**下，每處理一位病人會開啟及關閉報告編輯器，過程中請避免操作 RIS 視窗
> - **TAH 模式**下，報告編輯器會保持開啟，程式會自動切換病人，請勿手動切換
> - **螢幕擷取模式的 PACS** 同樣需要視窗保持可見，不能最小化
> - 程式會自動防止電腦進入睡眠或螢幕保護程式，但請避免手動讓電腦進入睡眠

### PACS 帳號密碼輸入

若使用 API 方式的 PACS（如台大醫院及各分院），批次開始時會跳出帳號密碼輸入視窗：

1. 輸入您的 PACS 登入帳號與密碼
2. 按「Submit」提交
3. 帳號密碼會被記住，同一批次的後續病人不需要重新輸入
4. 若登入失敗（例如密碼錯誤），程式會自動清除快取並再次詢問

> 若不想輸入，可按「Cancel」取消，但批次處理將會中止。

### 報告草稿檢視器

點擊「Open Draft Viewer」可開啟獨立的草稿檢視視窗，功能包括：

- 瀏覽所有已產生的報告草稿
- 檢視、編輯報告內容
- **Paste to RIS**：將草稿貼回 RIS 報告編輯器
- 複製報告內容至剪貼簿
- 批次刪除草稿

> 💡 關閉草稿檢視器視窗不會關閉主程式，也不會影響正在進行的批次處理。視窗只是隱藏，下次點擊「Open Draft Viewer」即可重新顯示。

### 修改設定

點擊右上角 ⚙ 圖示可開啟設定視窗，隨時可修改：

- RIS / PACS 系統切換
- AI 服務商、模型、API Key 更換
- 重試次數、超時時間
- 貼入模式（覆蓋 / 附加 / 插入 / 跳過 / 保留）
- 強制重新產生（忽略已有草稿）

### 自訂判讀提示詞（Skill Management）

點擊右上角 🎛 圖示可開啟判讀技能管理視窗，針對每個判讀技能個別設定提示詞：

- 從下拉選單選擇判讀技能（例如：ChestXR、MammographyMG 等）
- 勾選「使用內建提示詞」使用程式預設；取消勾選後可自行編輯
- 分為「系統提示詞」與「使用者提示詞」兩個標籤頁
- **AI 輔助生成**：點擊「AI Generate」可讓 AI 根據現有提示詞自動產生改版建議，預覽後可選擇接受或捨棄
- 點擊「Save」儲存後立即生效（儲存至 `skills.yaml`，不需重啟程式）

> ⚠️ 自訂提示詞僅影響 AI 的判讀風格與格式，不會更動程式的其他行為。若設定錯誤導致報告品質變差，可勾回「使用內建提示詞」立即恢復預設。

### 超時時間設定說明

設定中有兩個超時相關選項，可根據您的網路環境和 AI 模型調整：

| 設定項目 | 預設值 | 說明 |
|---|---|---|
| AI 請求超時（秒） | 60 | 單次 AI API 請求的等待時間。網路較慢或使用較大模型時可適當調高 |
| AI 整體期限（秒） | 90 | 處理每位病人的 AI 總時限（含重試）。若此時間內未完成，該病人會標記為超時失敗 |

> 💡 **調整建議**：若使用回應較慢的模型（如 GPT-4o）或網路不穩定，建議將 AI 請求超時設為 90–120 秒，AI 整體期限設為 120–180 秒。若使用快速模型（如 GPT-4o-mini、Gemini Flash），預設值通常足夠。

> ⚠️ 修改 RIS 或 PACS 設定後，已載入的病人清單會失效，需重新點擊「Load Worklist」。

### 貼入模式說明

| 模式 | 行為 | 適用情境 |
|---|---|---|
| 覆蓋 (Overwrite) | 取代 RIS 報告欄位原有內容（預設） | 一般使用 |
| 附加 (Append) | 將報告加在原有內容後方 | 需保留原內容並加上 AI 報告 |
| 插入 (Prepend) | 將報告加在原有內容前方 | 需將 AI 報告放在最前面 |
| 保留 (Keep) | 若 RIS 欄位已有內容則跳過，不貼入 | 不想覆蓋已手動輸入的報告 |
| 跳過 (Skip) | 不貼入 RIS | 只想產生草稿、稍後手動貼入 |

> 💡 「保留」模式下，若報告欄位已有內容，該病人會被標記為跳過，您可稍後在草稿檢視器中手動貼入。

### 強制重新產生

預設情況下，若某位病人已有先前產生的草稿，程式會直接使用舊草稿貼入 RIS，不會重新呼叫 AI。若您希望所有病人都重新由 AI 產生報告，請開啟「強制重新產生」選項。

## 常見問題

### Q: 批次處理時可以做其他事嗎？

建議盡量不要。程式需要操作 RIS 和 PACS 視窗，若您在批次處理期間：
- 切換到其他視窗，可能導致程式找不到正確的視窗
- 操作 RIS 或 PACS，可能干擾程式的自動操作
- 讓電腦進入睡眠，會中斷處理程序

若必須做其他事，建議暫停批次處理（按「Stop」），完成後再重新開始。

### Q: 為什麼有些病人處理失敗？

常見失敗原因及對應方式：

| 原因 | 說明 | 處理方式 |
|---|---|---|
| 找不到影像 | PACS 中沒有該病人的影像，或影像尚未載入完成 | 確認 PACS 中該病人確實有影像；螢幕擷取模式需等影像載入 |
| AI 超時 | AI 服務回應超過 90 秒 | 可能是網路問題或 AI 服務忙碌，可重試 |
| 沒有對應技能 | 該檢查項目不在支援範圍內 | 這是正常跳過，非錯誤 |
| 報告欄位已佔用 | 「保留」模式下 RIS 欄位已有內容 | 正常跳過；切換為覆蓋模式可覆蓋 |
| RIS 視窗無法操作 | 視窗被遮擋、最小化或關閉 | 確保 RIS 視窗保持可見 |
| PACS 登入失敗 | 帳號或密碼錯誤 | 重新輸入正確的帳號密碼 |
| 網路中斷 | 無法連線至 AI 服務或 PACS | 確認網路連線後重試 |

失敗的病人會顯示在完成清單中，可手動重試。

### Q: 可以中途停止嗎？

可以。點擊「Stop」按鈕即可停止批次處理。已完成和已失敗的病人數量會顯示在畫面上。已產生的草稿不會遺失。

### Q: API Key 安全嗎？

API Key 僅儲存於本機設定檔中，不會上傳至任何伺服器。

> ⚠️ **請妥善保管設定檔**。設定檔中的 API Key 和 PACS 帳號密碼皆為**明碼儲存**（未加密），若設定檔遭他人取得，可能導致您的 AI 服務額度被盜用或 PACS 帳號被冒用。建議：
> - 不要將設定檔分享給他人
> - 不要將設定檔上傳至公開的雲端空間或程式碼庫
> - 離開電腦時鎖定螢幕

### Q: 如何查看 Log 找出錯誤原因？

程式會自動記錄詳細的執行日誌，儲存在 `logs` 資料夾中。若批次處理出現問題，可以透過以下方式排查：

1. 開啟程式所在資料夾中的 `logs` 子資料夾
2. 找到最新的日誌檔案（檔名格式為 `conductor.log`）
3. 以文字編輯器開啟，搜尋 `ERROR` 或 `WARNING` 關鍵字

日誌分為四個等級：

| 等級 | 說明 |
|---|---|
| DEBUG | 詳細除錯資訊（預設不顯示於介面，僅寫入檔案） |
| INFO | 一般執行資訊 |
| WARNING | 警告（不影響執行但需注意） |
| ERROR | 錯誤（導致處理失敗的原因） |

> 💡 若想查看更詳細的除錯資訊，可在設定中將日誌等級改為 `DEBUG`。日誌檔案超過 5MB 時會自動輪替，最多保留 3 份備份。

### Q: 支援哪些檢查項目？

程式會根據檢查名稱中的關鍵字自動判斷對應的判讀技能，目前支援以下類別：

| 判讀技能 | 支援的檢查關鍵字 | 說明 |
|---|---|---|
| 胸部 X 光 | chest、cxr、thorax、lung、heart、cardiac、rib、pneumonia、pleural | 胸部及心肺相關 X 光 |
| 脊椎 X 光 | spine、cervical、lumbar、thoracic、scoliosis、vertebra、l-spine、c-spine、t-spine、sacrum、coccyx | 頸椎、胸椎、腰椎、薦椎、尾椎及側彎相關 X 光 |
| 腹部 X 光 | abdomen、abd、kub、abdominal | 腹部及泌尿系統 X 光 |
| 骨骼 X 光 | shoulder、knee、elbow、humerus、hand、finger、wrist、forearm、foot、ankle、hip、pelvis、femur、tibia、fibula、bone survey 等 | 四肢及骨骼相關 X 光（含肩、膝、肘、手、腕、足、踝、髖、骨盆等） |
| 顱骨 X 光 | skull、paranasal、sinus、waters view | 顱骨及鼻竇相關 X 光 |
| 面部 X 光 | nasal bone、mandible、orbit、zygomatic、facial、tmj、panfacial | 面部骨骼、鼻骨、下顎骨、眼眶、顴骨、顳顎關節等 |
| 骨齡 X 光 | bone age | 骨齡評估 |
| 頸部 X 光 | neck | 頸部軟組織 X 光 |
| 乳房攝影 | mammograph、mammogram、mg（影像類型判斷） | 乳房攝影（BI-RADS 2025 詞彙，含 BI-RADS 分類） |
| 通用判讀 | （以上皆不匹配時） | 所有未分類的影像檢查（支援所有造影類型，包括 CT、MR 等） |

> 💡 判斷方式為檢查名稱中的關鍵字比對（不分大小寫），若以上關鍵字都不匹配，則自動使用「通用判讀」模式。若病人的檢查項目不在任何技能範圍內，該病人會被自動跳過。

### Q: 需要網路連線嗎？

需要。AI 報告產生必須連線至 AI 服務商。PACS 若使用 API 方式也需要網路。若使用螢幕擷取方式的 PACS，影像已在本地 PACS 視窗中，但 AI 服務仍需網路。

### Q: 如何更新程式？

點擊主畫面的「Check for Updates」按鈕即可檢查是否有新版本。若有新版本，按「Update」即可自動下載並更新。

> ⚠️ 批次處理進行中無法更新，請先停止批次處理再進行更新。

### Q: 草稿會保留嗎？

會。AI 產生的報告草稿會自動儲存，即使關閉程式也不會遺失。下次啟動後可在草稿檢視器中繼續檢視或貼入 RIS。

### Q: 彈出視窗會不會干擾批次處理？

不會。程式內建彈出視窗自動處理功能，會自動偵測並關閉 RIS 彈出的提醒視窗，例如：

- **NTUH**：「注意」對話框（報告已被佔用）、「無更多待處理檢查」提示
- **TAH**：多筆檢查選擇對話框、「查無影像」提示、確認離開對話框

這些彈出視窗會被自動處理，不會中斷批次流程。

### Q: 為什麼載入清單後有些病人沒有檢查號？

這通常表示 RIS 工作清單中沒有顯示「照會單號」欄位。請確認 RIS 工作清單中有顯示照會單號（Accession Number）欄位，程式需要此欄位來識別每位病人。

### Q: NTUH 和 TAH 模式有什麼差別？

| | NTUH 模式 | TAH 模式 |
|---|---|---|
| 報告編輯器 | 每位病人開啟及關閉 | 保持開啟，自動切換病人 |
| 批次報告儲存 | 貼入後自動儲存 | 不儲存（結果送至草稿檢視器） |
| PACS 選項 | API 或螢幕擷取 | 螢幕擷取 |
| 彈出視窗 | 自動處理 3 種 | 自動處理 3 種 |
| 草稿檢視器貼入 | 僅貼入，不儲存 | 僅貼入，不儲存 |

### Q: 為什麼 PACS 帳號密碼又要重新輸入？

PACS 帳號密碼會在同一批次中記住，不需重複輸入。但以下情況需要重新輸入：

- **新批次開始**且上次未勾選記住密碼
- **登入失敗**（密碼錯誤或帳號過期）時，程式會自動清除快取
- **更換 PACS 來源**（例如從台大醫院切換到分院）時，因為不同醫院的帳號不同

### Q: 可以同時開啟多個 RIS Conductor 嗎？

不建議。多個實例可能會同時操作同一個 RIS 視窗，導致衝突和錯誤。

### Q: 草稿檢視器的「Paste to RIS」怎麼用？

1. 在草稿檢視器中選擇要貼入的報告
2. 點擊「Paste to RIS」按鈕
3. 程式會自動導航到對應的病人並貼入報告（僅貼入，不儲存或送出）
4. 貼入成功後草稿狀態會更新為「已貼入」

> ⚠️ 使用「Paste to RIS」前，請確保 RIS 視窗已開啟且可見。
